# Boas Práticas de Documentação Técnica

## Estrutura

- Mantenha documentação separada por público (usuário, dev, API).
- Inclua seções claras: introdução, instalação, uso e exemplos.
- Utilize Markdown para padronizar formatação.

## Atualização Contínua

- Atualize a documentação a cada nova versão.
- Automatize a geração de docs quando possível (ex: Swagger, Docusaurus).
- Marque versões obsoletas e evite duplicidade de informações.

## Clareza e Linguagem

- Escreva de forma direta e objetiva.
- Use exemplos reais e funcionais.
- Inclua diagramas e fluxos quando necessário.

## Integração com o Código

- Documente endpoints, funções e módulos críticos.
- Use comentários de documentação (docstrings, JavaDoc, etc.).
